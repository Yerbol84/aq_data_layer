## 📚 Новые документы

### KEY_DECISIONS.md

Создан новый документ **`doc/architecture/KEY_DECISIONS.md`**, который содержит:

1. **Философия "Тонкого клиента"**
   - Обоснование решения
   - Преимущества
   - Примеры реализации

2. **Архитектурные паттерны**
   - Единая схема (Single Source of Truth)
   - Унифицированные константы (VersionedStorageContract)
   - PostgreSQL-оптимизированные реализации

3. **Multi-tenancy**
   - Выбор подхода (tenant_id колонка)
   - Отвергнутые альтернативы
   - Преимущества решения

4. **Storage Types**
   - Direct, Versioned, Logged
   - Use cases для каждого типа
   - Структура таблиц

5. **RPC Protocol**
   - Обоснование выбора RPC вместо REST
   - Формат запросов

6. **Буферизация**
   - Назначение и use cases

7. **Тестирование**
   - Три уровня тестов
   - Инфраструктура

8. **Производительность**
   - Оптимизации PostgreSQL
   - Бенчмарки

9. **Безопасность**
   - Row Level Security (RLS)
   - SQL Injection Protection

10. **Deployment**
    - Автоматическое создание схемы

**Цель документа:** Сохранить архитектурные решения и их обоснования для будущих разработчиков.
