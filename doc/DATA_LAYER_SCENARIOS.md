# DATA_LAYER_SCENARIOS.md — Сценарии использования Data Layer

**Версия:** 1.0  
**Дата:** 2026-04-30

Data layer закрывает все потребности платформы в хранении данных.  
Клиент не знает о реализации — только о модели и репозитории.

---

## Режимы хранения

### 1. Direct — простой CRUD

**Когда использовать:** объекты без истории изменений.  
**Модели:** `AqStudioProject`, `GraphRunState`

**Сценарии:**
- Создать / обновить / удалить проект
- Найти по ID
- Список с фильтрацией и пагинацией
- Soft delete (пометить удалённым) vs Hard delete (физически удалить)
- Восстановить soft-deleted объект

### 2. Versioned — версионирование

**Когда использовать:** объекты с жизненным циклом версий (draft → published → archived).  
**Модели:** `WorkflowGraph`, `InstructionGraph`, `PromptGraph`

**Сценарии:**
- Создать черновик (draft)
- Редактировать черновик (новый node в дереве версий)
- Опубликовать версию (minor / major / patch)
- Создать ветку (branch) для экспериментов
- Смержить ветку в main
- Откатиться к предыдущей версии
- Выдать доступ другому пользователю (grant)
- Отозвать доступ (revoke)
- Получить историю версий

### 3. Logged — CRUD с аудит-логом

**Когда использовать:** критичные данные требующие полной истории операций.  
**Модели:** `WorkflowRun`

**Сценарии:**
- Создать запись (автоматически логируется)
- Обновить запись (diff сохраняется в лог)
- Удалить запись (лог сохраняется)
- Получить лог изменений конкретной записи
- Rollback к предыдущему состоянию

### 4. Artifact — бинарные файлы

**Когда использовать:** файлы с метаданными (изображения, документы, экспорты).

**Сценарии:**
- Загрузить файл с метаданными
- Скачать файл по ID
- Найти файлы по метаданным
- Удалить файл

### 5. Knowledge — документы + векторный поиск

**Когда использовать:** текстовые документы для семантического поиска (RAG).

**Сценарии:**
- Добавить документ в knowledge base
- Семантический поиск по запросу
- Найти похожие документы
- Удалить документ из базы знаний

## Сценарии подключения

### A. Удалённый сервер (production)
```dart
await IDataLayer.initialize(endpoint: 'http://data-service:8765');
// Handshake → получить список коллекций → работать
```

### B. Локальный буфер (offline-first UI)
```dart
await Vault.connect('http://data-service:8765', useBuffer: true);
// Записи идут в локальный буфер → flush при синхронизации
```

### C. In-memory (тесты, dev без сервера)
```dart
// Автоматический fallback если сервер недоступен
await Vault.connect('http://localhost:8765', failFast: false);
```
